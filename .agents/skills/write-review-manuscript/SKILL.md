# Re:VIEW原稿を書く

このリポジトリのRe:VIEW 5.9原稿を、既存の構造と有効な記法を保って執筆する。

## 作業手順

1. 対象の `.re` と、その前後の章を `articles/catalog.yml` で確認する。
2. 記法を追加・変更する前に [references/review-format.md](references/review-format.md) の該当項目を読む。段落だけを書く場合も「最重要ルール」と「段落」を読む。
3. 既存原稿の語調・見出し階層・識別子の付け方を優先し、Markdown記法を混ぜない。
4. 画像、リスト、表、脚注には章内で一意かつ意味の分かる識別子を付け、本文からRe:VIEWの参照命令で参照する。
5. 変更後はブロックの `//}`、インライン命令の `}`、参照先識別子、リスト行頭の空白を点検する。
6. 原稿を編集した場合は、可能ならリポジトリルートで `npm test` を実行し、HTML変換が成功することを確認する。依存関係が未導入なら、その旨を報告する。

## 判断基準

- 本文の改行と段落分割を区別する。空行がなければ、複数行でも同じ段落になる。
- 見出しは現在位置より1段だけ深くする。章は `=`、節は `==`、項は `===` を基本とし、不必要に階層を飛ばさない。
- 箇条書きは行頭に半角空白を置く。`* item` ではなく ` * item` と書く。
- コードやコマンドをMarkdownのバッククォートフェンスで囲まない。用途に合うRe:VIEWブロック命令を使う。
- 図のファイルは `articles/images/` に置く。このリポジトリでは `//image[id][caption][scale=...]` を使用できる。
- 見た目を作るためだけの空行、強制改行、生データ埋め込みを濫用しない。
- 記法が不明な場合は推測で独自タグを作らず、Re:VIEW 5.9のフォーマットガイドとこのテンプレートの実装を確認する。
